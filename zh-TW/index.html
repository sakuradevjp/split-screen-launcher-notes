1. 背景：為什麼 Android 上的「應用程式配對」很重要

Android 從 7.0（Nougat，2016 年）就開始支援分割畫面多工。但它始終沒有在 OS 層面支援過 把兩個應用的組合儲存起來並同時啟動。三星早在 2017 年就把這項功能以「應用程式配對」的名義加入 One UI；從 Galaxy 換到 Pixel 的使用者幾乎第一時間就會注意到它消失了。

變通辦法是安裝第三方啟動器，把兩個步驟（打開應用 A、把應用 B 開到相鄰視窗）自動化。Google Play 上這類應用有幾十個，但生態比表面看起來亂——每一次 Android 更新都會改變分割畫面的呼叫方式，而大多數應用都沒有跟進。

2. Split Screen Launcher 實際能做什麼

Split Screen Launcher 是一款專注於單一用途的工具。你選兩個應用、幫它們取個名字（「通勤」、「工作」、「YouTube + X」），這組配對就會以一列的形式出現在應用中。點一下，兩個應用就會並排打開。

整個功能真的就這些。沒有內建瀏覽器、沒有浮動小工具、沒有通知列整合。APK 大約 2.8 MB，比多數應用圖示還小，而且不會在背景執行任何東西。

測試時我們使用的配對範例：

  • Google 地圖 + Spotify — 開車
  • YouTube + X（Twitter） — 第二螢幕觀看
  • Chrome + Google Keep — 一邊查資料一邊筆記
  • Gmail + Slack — 工作訊息分流
  • 計算機 + Chrome — 多分頁比價

有個細節容易被漏看，值得特別提：配對資料可以匯出成備份檔。聽起來不起眼，但同類競品常常把配對存在應用私有目錄裡，完全沒有匯出管道——換手機就得手動重建整份配對。我們從一開始就把備份/還原做成一級功能，正是為了讓 Pixel 換 Pixel 時幾秒就能恢復所有配對。

另一個值得先講清楚的設計取捨：本應用不會建立主畫面捷徑。每個配對都只存在於應用內部。這是一種取捨。代價是要啟動配對就得先打開應用——相較於以捷徑為主的競品多一步。好處有兩個：主畫面不會被配對圖示塞爆；應用不會被目前啟動器（Pixel Launcher、Nova 等）的捷徑處理方式綁架。後者比聽起來更重要——正如第 3 節所說，以捷徑意圖為基礎的啟動器正是 Android 15、16 切換時最容易壞掉的那一類。把一切留在應用內，就可以把那一整類故障隔離在外。

適合你，如果你……

  • 經常把同兩個應用搭配使用（地圖+音樂、電郵+聊天）

  • 比起應用內管理，更偏好主畫面捷徑

3. Android 15/16 難題 — 為何大多數競品失效

這是有趣的部分，也是大多數同類應用不再能用的原因。

歷史上，分割畫面啟動器一直依賴少數幾個 OS 級 API，而 Google 一直在逐步收緊或移除。每一次 Android 新版本都在削弱第三方應用用來並排啟動兩個應用的手段。Android 15 和 16 尤其關閉了最常見的路徑。shell 層級的變通也以類似方式失敗。

白話來說：第三方應用過去用來把手機切到分割模式的「小撇步」，在 Android 15 與 16 上已經失效。這個類別裡的大多數應用都沒推出修正版。

結果是 Google Play 上大多數分割畫面工具——包括這個類別下載量最高的那款——在近期 Pixel 裝置上都已無法正常運作。這些應用的使用者評論裡，從 2025 年底開始就堆滿投訴。

Split Screen Launcher 透過另一條路徑，在 Android 15 與 16 上依舊能穩定進入分割畫面。具體的機制我們不會公開 — 要找到一套能跨 OS 版本運作的組合花了不少試驗，公開做法等於把它交給每一個已經壞掉的競品。能說的是，我們的發布紀錄顯示，在 Android 12、12L、14、15、16 的每次切換中都有持續維護，而且我們打算繼續保持這個節奏。

仍然做不到的事：有些應用在 manifest 中宣告 android:resizeableActivity="false" — 原生相機與 Google Files 就是代表 — OS 會硬性阻擋它們進入分割畫面，不論怎麼啟動都一樣。就連三星內建的應用程式配對也無法把它們並排起來。這不是應用程式的限制，也不是任何第三方工具可以解決的。

白話來說：這些應用已經告訴 Android「我拒絕在分割模式下執行」，任何第三方工具 — 甚至三星自己的 — 都無法覆蓋這個決定。

4. 權限與隱私

在 Android 12L 以上，應用不需任何執行階段權限，也不需要無障礙服務。所有操作都透過標準 Activity 旗標完成。

在 Android 9 到 12，應用確實需要無障礙服務權限，因為這是唯一可用的 API。Play Store 商店頁面明確說明這項服務僅用來觸發分割畫面。Google Play 顯示的權限清單短到可以在一個畫面看完 — 沒有位置、沒有通訊錄、沒有儲存空間，而且（特別是）沒有網路。對 Play Store 的這個角落來說，這已經算是異類。

白話來說：舊版 Android 沒有官方 API 可以觸發分割畫面，因此無障礙服務是當時唯一可行的路徑。這是遺留限制，不是應用拿來讀你螢幕的藉口。

作為對照，Play Store 上多數免費分割畫面應用都內建廣告 SDK（AdMob、Unity、IronSource），這表示它們附帶 INTERNET 權限及伴隨而來的追蹤器組。我們選擇改用最小化的訂閱來營收 — 一方面是為了讓權限足跡保持乾淨；另一方面是因為把廣告 SDK 塞進一個 2.8 MB 的工具，體積會膨脹超過三倍。

5. 與其他分割畫面應用的比較

比較之前，先勾勒整個版圖。「應用程式配對」類功能大致有幾種做法：

  • OEM 專屬實作 — 三星的應用程式配對（2017 年 One UI 起），以及小米、OPPO、舊 LG 的變體。只能在各家自家裝置上運作。
  • 第三方工具 — 專注在這一個用例的獨立應用。多數建立在近年 Android 收窄或移除的 OS 級 API 之上，因此這類應用數量已經明顯減少。
  • 自動化類做法 — Tasker 等工具可以用腳本做分割啟動，但它們依賴與專用應用相同的底層基礎，因此繼承了相同的故障。此外還得使用者自己手動為每組配對撰寫流程。

6. 價格

免費版會對可儲存的配對數量設上限，讓使用者在決定是否符合自己使用習慣前，就能完整體驗整個流程。核心功能 — 建立配對、啟動、備份/還原 — 都無需付費就能使用。

Pro 訂閱依地區約為 每月 $1，可解除配對上限，並新增資料夾整理及圖示顏色自訂。一般的 Google Play 訂閱，可隨時取消。

作為參考：同類的 CCS Software 應用在菲律賓當地的進階版價格據回報約 $13，「Split Screen Easy Multitasking」則為每週 $18.99。$1/月大約是最接近的付費競品的十分之一。

7. 總結 — 適合哪些人

這款應用適合誰？ 誠實評價

我們不認為這款應用適合所有人。如果你本來就沒有固定的雙應用習慣（地圖+音樂、電郵+聊天等），你大概不會常打開它。但如果你使用原生 Android，又從換到 Pixel 之後就一直懷念 Galaxy 的應用程式配對，這款應用就是為你做的 — 我們也努力讓它成為把該體驗找回來、最少妥協的方式。無廣告立場與 $1/月 的定價都是刻意的選擇，我們相信這對這個類別而言是對的。

優點

  • 在 Android 15 與 16 上真的能用
  • 所有方案皆無廣告
  • 2.8 MB，無背景服務
  • Android 12L+ 不需要無障礙權限
  • 完全未宣告網路權限
  • Pro 方案約 $1/月，比競品便宜 10 倍
  • 備份 / 還原，適合換機
  • 15 種語言在地化

缺點

  • 從點擊到分割畫面出現之間，會有大約 1 秒的可感延遲。這多半是 OS 端的代價 — Android 12L+ 進入分割模式涉及多次系統內部的任務切換，所以這個類別的任何應用都會落在 0.8–1.2 秒之間。不是 bug，但值得先讓使用者知道
  • 沒有主畫面捷徑、沒有浮動小工具、沒有手勢綁定、沒有 Tasker 整合 — 啟動配對需要先打開應用。這是刻意的設計決定（見第 2 節），但確實多一次點擊
  • 建立配對的清單會無分類列出所有已安裝應用，一旦裝置上超過約 60 個應用就會變得有點繁瑣
  • 被 OS 標記為不可調整大小的應用（Google Files、原生相機）依然無法分割 — 不是應用的錯，但最好在撞到限制前就知道
  • 獨立開發者，所以如果你在非 Pixel 的 ROM 上遇到邊緣案例，不要期待當天修好
